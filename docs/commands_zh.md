# 命令参考

## 连接管理

### `sshgo add <name>`

添加新的 SSH 连接配置。

| 参数 | 说明 | 必填 |
|------|------|------|
| `<name>` | 连接别名（小写字母、数字、连字符，不能以连字符开头） | 是 |
| `--host` | 服务器 IP 或域名 | 是 |
| `--port`, `-p` | SSH 端口（默认 22） | 否 |
| `--user`, `-u` | SSH 用户名 | 是 |
| `--group` | 所属分组名称 | 否 |
| `--identity-file`, `-i` | 私钥文件路径 | 否 |

```bash
sshgo add my-server --host 192.168.1.10 --user deploy -p 2222 --group prod

# 使用密钥认证
sshgo add prod --host 10.0.0.1 --user deploy -i ~/.ssh/id_ed25519_prod
```

### `sshgo show <name>`

显示连接配置的详细信息（含跳板机、端口转发等）。

```bash
sshgo show my-server
```

### `sshgo edit <name>`

打开 `~/.sshgo/config.yaml` 进行手动编辑。

```bash
# 编辑指定连接（打开完整配置文件，光标不一定在对应行）
sshgo edit my-server

# 编辑完整配置文件
sshgo edit
```

### `sshgo delete <name>`

删除指定的连接配置（同义词：`rm`、`del`）。

```bash
sshgo delete my-server
sshgo rm my-server
```

## 查询

### `sshgo list`

列出所有连接配置。

| 参数 | 说明 |
|------|------|
| `--group` | 按分组过滤 |
| `--sort` | 排序方式：`name` |
| `--format`, `-f` | 输出格式：`table`（默认）、`json` |

```bash
sshgo list                  # 列出所有
sshgo list --group prod     # 只显示 prod 分组
sshgo list --sort name      # 按名称排序
sshgo list --format json    # JSON 输出
sshgo ls                    # 别名
```

## 连接

### `sshgo connect <name>`

连接到已保存的 SSH 会话。实际执行 `exec ssh` 替换当前进程。

| 参数 | 说明 |
|------|------|
| `<name>` | 连接别名 |
| `--recent` | 从连接历史中选择 |

```bash
sshgo connect my-server
sshgo my-server           # 快速模式（等价）
sshgo connect --recent    # 从历史选择
```

## 分组管理

### `sshgo group list`

列出所有分组及其包含的连接数。

### `sshgo group add <name>`

添加新分组。

| 参数 | 说明 |
|------|------|
| `--description` | 分组描述 |

```bash
sshgo group add prod --description "Production servers"
```

### `sshgo group delete <name>`

删除分组（不影响该分组下的连接配置）。

```bash
sshgo group delete prod
```

## 跳板机

### `sshgo add-jump <name>`

为指定连接添加跳板机（支持多层跳板链）。**会替换已有的跳板机链**——如需查看或修改现有链，请使用 `sshgo show <name>` 与 `sshgo edit`。

| 参数 | 说明 |
|------|------|
| `<name>` | 目标连接别名 |
| `--jump` | 跳板机地址（可重复使用） |
| `--identity-file` | 第 N 个 `--jump` 对应的私钥文件（按位置配对，可重复） |

跳板机地址格式：`[user@]host[:port]`，user 默认为 root。

```bash
# 单层跳板
sshgo add-jump db-server --jump bastion

# 多层跳板链
sshgo add-jump db-server --jump bastion --jump gateway

# 带用户和端口
sshgo add-jump db-server --jump deploy@bastion:2222

# 为每一层跳板机指定独立密钥（按位置与 --jump 配对）
sshgo add-jump db-server \
  --jump admin@bastion1:22 -i ~/.ssh/id_bastion1 \
  --jump ops@bastion2:22   -i ~/.ssh/id_bastion2
```

## 端口转发

### `sshgo forward add <name>`

为指定连接添加端口转发规则。

| 参数 | 说明 |
|------|------|
| `<name>` | 连接别名 |
| `--local`, `-L` | 本地端口转发（格式：`local_port:remote_host:remote_port` 或 `local:remote`） |

```bash
# 简写（remote_host 默认为 localhost）
sshgo forward add my-server -L 8080:80

# 完整格式
sshgo forward add my-server -L 8080:db-host:5432
```

### `sshgo forward list`

列出所有端口转发规则。

```bash
sshgo forward list
```

## 批量执行

### `sshgo exec <pattern> <command>`

对多台服务器并行执行命令。

| 参数 | 说明 |
|------|------|
| `<pattern>` | 匹配模式：通配符、逗号分隔列表 |
| `<command>` | 要执行的命令 |
| `--group` | 对指定分组内所有服务器执行 |

```bash
# 通配符匹配
sshgo exec 'web-*' "uptime"

# 逗号分隔的多台
sshgo exec "web-1,web-2,db-1" "df -h"

# 按分组执行
sshgo exec --group prod "systemctl status nginx"
```

输出格式：

```
=== web-1 ===
 10:30:45 up 42 days, 2 users

=== web-2 ===
 10:30:45 up 15 days, 1 users

Total: 2 succeeded, 0 failed
```

**注意**：批量执行使用 SSH 默认认证（密钥、Agent）。需要密码认证的服务器可能无法交互输入密码。

## 导入/同步

### `sshgo import`

从 OpenSSH 配置文件导入连接。

| 参数 | 说明 |
|------|------|
| `--file`, `-f` | 源文件路径（默认 `~/.ssh/config`） |
| `--overwrite`, `-w` | 覆盖已有同名配置 |

```bash
sshgo import
sshgo import --file /path/to/custom-ssh-config
sshgo import --overwrite
```

支持解析：Host, HostName, User, Port, IdentityFile, ProxyJump, Include 指令。

### `sshgo sync`

将 sshgo 配置导出为 OpenSSH 兼容格式，写入 `~/.ssh/config`。

| 参数 | 说明 |
|------|------|
| `--dry-run` | 预览输出，不写入文件 |

```bash
sshgo sync --dry-run    # 预览
sshgo sync              # 写入 ~/.ssh/config
```

**行为说明**：
- 自动备份原 `~/.ssh/config` 为 `~/.ssh/config.sshgo.bak`
- 保留 `~/.ssh/config` 中非 sshgo 生成的内容
- 替换 sshgo 生成的区块（标记为 `# Generated by sshgo`）

## 其他

### `sshgo ping <name>`

测试到目标服务器的 SSH 连通性。

```bash
sshgo ping web-server
```

输出：`✓ user@host:port - Connected in 1.234s` 或 `✗ user@host:port - Connection failed (...)`

超时时间：5 秒（BatchMode 非交互模式）。

### `sshgo history`

查看连接历史记录。

```bash
sshgo history
```

### `sshgo doctor`

检查所有 profile（包括跳板机）的问题：缺失的密钥文件、权限过宽的密钥文件、以及其他 `Profile.Validate` 能发现的问题。如发现问题则以非零状态退出。

适用于手改 `~/.sshgo/config.yaml` 之后、导入外部 SSH config 之后、或 sshgo 感知之外的 `chmod` 变更之后。

```bash
sshgo doctor
# [OK] All profiles healthy.

# 有问题时的示例：
sshgo doctor
# Found 1 issue(s):
#   - my-server: identity file /home/me/.ssh/leaky has insecure permissions (0644); run: chmod 600 /home/me/.ssh/leaky
```

### `sshgo completion [bash|zsh|fish]`

生成 Shell 自动补全脚本。

```bash
# Bash
source <(sshgo completion bash)

# Zsh
sshgo completion zsh > "${fpath[1]}/_sshgo"

# Fish
sshgo completion fish | source
```
